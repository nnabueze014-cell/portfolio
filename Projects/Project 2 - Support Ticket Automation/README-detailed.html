Problem Statement

Many service-based technical companies still rely on informal channels such as calls and messages to receive customer requests. This often leads to poor tracking, missed SLAs, delayed execution, and lack of accountability across teams.

This project addresses that gap by designing a structured, automated system that ensures customer requests are acknowledged, tracked, executed, and resolved with full visibility.

System Architecture Overview

The solution is designed using a three-layer architecture:

  • Frontend Layer (Customer Support / Front Desk)
  • Automation & Integration Layer (System Bridge)
  • Operations & Execution Layer (Backend Unit)

1. Frontend Layer ‚Äì Customer Support (Freshdesk)

The system begins with a customer-facing support portal built using the Freshdesk widget, embedded on a demo website. This serves as the official entry point for all service requests.

  • Customers submit service requests via the embedded widget
  • Requests are categorized by service type and priority
  • Preferred support method (call or email) is captured
  • First response and resolution SLAs are automatically applied
  • Automations monitor SLA compliance and escalate when breached

This layer focuses on customer communication, acknowledgment, and service-level performance.

2. Automation & Integration Layer ‚Äì System Bridge (Zapier)

To eliminate manual handoffs between support and operations, an automation layer was implemented using webhooks and Zapier.

  • Freshdesk sends a webhook event immediately a ticket is created
  • Zapier listens for the event and processes the ticket data in real time
  • A corresponding task is created in the operations tool
  • Tasks are routed to the correct service pipeline (Electrical, Solar, CCTV, Home Automation, HVAC)
  • Priority levels, start dates, and due dates are mapped automatically

This ensures instant transition from customer request to operational action.

3. Operations & Execution Layer ‚Äì Backend Unit (ClickUp)

The backend execution layer is managed in ClickUp, where engineers and technicians carry out the work.

  • Support tickets become actionable operational tasks
  • Tasks are assigned to technicians with supervisors added as watchers
  • A separate execution SLA governs delivery timelines
  • SLA breaches trigger automatic status updates and alerts

This allows engineers to focus on execution while management maintains visibility and control.

SLA Monitoring & Escalation (End-to-End)

  • Front-desk SLA breaches escalate ticket priority and notify supervisors
  • Backend SLA breaches update task status and notify relevant stakeholders
  • Accountability is enforced from first customer contact to final resolution
